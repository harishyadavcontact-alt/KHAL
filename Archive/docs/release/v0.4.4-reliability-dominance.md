# v0.4.4 Reliability Dominance Slice

## Scope
1. Harden agent commit safety: commit accepts only an approved subset of dry-run proposals.
2. Add tri-readable API contract regression tests (`decision/evaluate`, `decision/override`, `agent dry-run/commit`).
3. Add workflow hygiene audit to block deprecated CI drift.
4. Add one-command main-branch release packaging with RC -> final tag flow.

## Exact Files

2. `apps/web/test/tri-readable-api-contract.test.ts`

8. `docs/release/v0.4.3-release-runbook.md`

## Commands
```bash
npm run ci:audit
npm --workspace @khal/web run test -- tri-readable-api-contract.test.ts
npm --workspace @khal/web run test
npm run quality:gate
npm run release:package:main
```

## Acceptance Gates
1. `tri-readable-api-contract.test.ts` is green.
2. Dry-run commit rejects out-of-proposal mutations (`400`).
3. Valid dry-run subset commit succeeds and marks row `COMMITTED`.
4. Workflow audit fails on deprecated/unknown workflow files or `pnpm` usage in CI.
5. `quality:gate` emits `artifacts/quality/quality-gate.json` with `status=passed`.
6. `release:package:main` runs only on clean `main`, then creates local `rc` and final tags.

## Release Sequence
1. Merge on `main`.
2. Run `npm run release:package:main`.
3. Push tags (or rerun script with `-PushTags`) once CI is green.
