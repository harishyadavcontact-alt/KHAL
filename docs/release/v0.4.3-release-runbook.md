# v0.4.3 Release Runbook

## Release Type
GitHub release only. No auto-deploy.

## Preconditions
1. `main` branch is green on required checks.
2. Quality artifacts are produced under `artifacts/quality`.
3. UAT report is attached and approved.
4. Local workflow audit passes (`npm run ci:audit`).

## Required Gate Order
1. `npm ci`
2. `npm run typecheck`
3. `npm run build`
4. `npm --workspace @khal/web run test`
5. start server on 3010
6. `node scripts/smoke-routes.mjs --baseUrl=http://localhost:3010`
7. `node scripts/perf-smoke.mjs --baseUrl=http://localhost:3010`
8. `node scripts/qa-report.mjs`

## Release Steps
1. Trigger `Release` workflow (manual dispatch) with target tag `v0.4.3-rcN` or `v0.4.3`.
2. Confirm artifact upload: smoke/perf/qa JSON + changelog excerpt.
3. Publish GitHub release notes and verify attached artifacts.

## Main-Branch Packaging Shortcut
Use one command on a clean `main` branch to run the full local gate and create RC/final tags:

```bash
npm run release:package:main
```

Optional publish:

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File scripts/release-package-main.ps1 -RcTag v0.4.4-rc1 -FinalTag v0.4.4 -PushTags
```

## Rollback
1. Identify previous known-good tag.
2. Revert `main` to that tag via standard GitHub rollback process.
3. Re-run CI + smoke.
4. Verify SQLite migration compatibility before redeploying binaries.
