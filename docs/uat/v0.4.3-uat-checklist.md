# v0.4.3 UAT Checklist

## Preconditions
- Branch is synced with `main`.
- `npm ci`, `npm run typecheck`, `npm run build`, `npm --workspace @khal/web run test` are green.
- Web app is running on `http://localhost:3010`.

## Missionary Role Flow
1. Open `/war-gaming/source` and verify flow rail shows predecessor-first sequence.
2. Open `/war-gaming/domain` with incomplete predecessor data and verify risky CTA is blocked.
3. Verify block reason card includes exact missing items.
4. Confirm no-ruin failure blocks execution CTAs.

## Visionary Role Flow
1. Switch role to Visionary in war-gaming.
2. Jump directly to `/war-gaming/mission`.
3. Verify navigation is allowed but dependency warning is visible.
4. Verify hard doctrine gates still block execution.

## Mode-Specific War-Gaming
1. `source`: confirm source grammar fields and readiness behavior.
2. `domain`: confirm complex-domain monomodal block without override.
3. `affair`: confirm robustness/downside posture requirement.
4. `interest`: confirm convexity + max-loss requirement.
5. `craft`: confirm heap -> model -> framework -> barbell -> heuristic progression.
6. `lineage`: confirm dependency + blast/exposure view behavior.
7. `mission`: confirm hierarchy/dependency/no-ruin synthesis behavior.

## Tri-Readable Checks
1. Human-readable: identify next action from funnel/rail in under 1 second.
2. Agent-readable: call `/api/decision/evaluate`, verify deterministic payload for same input.
3. Compiler-readable: typecheck/build succeed without manual file generation steps.

## Evidence Rules
- Capture screenshot or logs for each failed step.
- Record blocker severity (`high`, `medium`, `low`).
- Fill `artifacts/quality/uat-report.json` and attach evidence IDs.

## Exit Criteria
- All high-severity issues are zero.
- No unresolved blocker in doctrine gates or execution controls.
- UAT report status marked `passed` with reviewer and timestamp.

## v0.4.5-R1 Triage + Doctrine
1. Open `/war-gaming/interest` with an incomplete protocol interest.
2. Verify `Triage` panel appears and top suggestion is deterministic across refresh.
3. Click one doctrine quick action (`max-loss`, `expiry`, `kill criteria`, or `barbell split`), verify readiness updates.
4. Verify block reasons still show exact missing items after each mutation.
5. Open `/dashboard`, verify `Next Actions` strip shows top 1-3 triage actions.
6. From dashboard strip, verify `Open` deep-link routes to `/war-gaming/{mode}?target={id}`.
7. Verify no-ruin blocked contexts only show recovery guidance and do not auto-mutate.
